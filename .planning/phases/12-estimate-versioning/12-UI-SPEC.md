## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding |
| sm | 8px | Compact row spacing, inline gaps |
| md | 16px | Default element spacing, cell padding |
| lg | 24px | Section padding, sheet header |
| xl | 32px | Major block gaps within sheet |
| 2xl | 48px | Sheet padding (top/bottom) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Version badge pill: 6px vertical / 10px horizontal (between xs and sm — matches existing `text-[10px]` pill pattern from `cost-breakdown.tsx` assumptions bar)
- Touch targets for checkboxes and inline-edit triggers: minimum 44px height on mobile
- Comparison table cells: 16px horizontal / 12px vertical (tighter than md to fit 4-column layout on mobile)

**Source:** 8-point scale is the project default. Exceptions derived from existing component measurements in `cost-breakdown.tsx` and `project-list.tsx`.

| Destructive | `--color-error` | `#ba1a1a` | Not used in this phase |
| Delta positive | Tailwind `text-red-500` | `#ef4444` | Cost increase delta values (positive, +$X) |
| Delta negative | Tailwind `text-green-500` | `#22c55e` | Cost savings delta values (negative, -$X) |
| Delta neutral | `--color-on-surface/40` | `#333333` at 40% opacity | Zero delta values |

**Accent reserved for:**
- "View history" trigger link text (matches primary link color convention)
- "Version N saved" banner border (`border-primary/20`) and background (`bg-primary/5`) — mirrors ICC disclaimer panel pattern in `cost-breakdown.tsx`
- Currently-selected version checkbox checkmark fill
- "Compare" button background when exactly 2 versions are selected (`bg-primary text-on-primary`)
- Active inline-edit input bottom border (`border-primary`) — matches `project-list.tsx` pattern

**Source:** Token values read from `src/app/globals.css`. Delta colors use Tailwind semantic red/green because there are no equivalent custom tokens in the design system. These two values are new to this phase and must not be used elsewhere.

**CostBreakdown dark surface note:** The `CostBreakdown` component uses an inverted dark surface (`bg-[#1a1a1a]`). The version badge and "Version N saved" banner added to it must respect this dark context: use `#ccff00` (same as existing accent uses in that component) for the badge pill background; use `text-[#f2f2f2]` for badge label text.

## Component Inventory

New components required this phase:

### 1. Version badge pill — inline in `CostBreakdown`
- Location: top of `CostBreakdown`, left-aligned, before the hero price block
- Visual: small pill `bg-[#ccff00] text-black` on dark surface; 6px/10px padding; `text-[10px] font-black uppercase tracking-widest font-headline`
- Content: "v{N} of {total}" (EN) / "v{N} de {total}" (ES)
- Trigger: only renders when `_persistedId` is present in the tool result (not shown for first in-flight estimate)
- Right of badge: "View history" trigger — `text-[10px] font-black text-[#ccff00] uppercase tracking-wider hover:underline cursor-pointer ml-3`

### 2. "Version N saved" banner — inline in `CostBreakdown`
- Location: below the version badge, above the hero price block; only shown immediately after a new version is created
- Visual: mirrors the ICC disclaimer panel — `bg-[#ccff00]/5 border border-[#ccff00]/20 rounded-lg px-4 py-3`
- Content: "Version {N} saved — [View history]" where "[View history]" is an inline trigger link
- Dismisses: on first user interaction or 8 seconds (whichever first); not stored in state beyond session

### 3. `VersionHistorySheet` — slide-out drawer
- Trigger: "View history" from badge or banner
- Width: full-width on mobile, 420px fixed on md+
- Slide direction: right edge
- Header: "Version History" (EN) / "Historial de versiones" (ES) — 16px bold; close button (X icon, 44px touch target)
- Body loading state: 3-row skeleton (see skeleton spec below)
- Two internal views: `"list"` and `"compare"` (see below)

### 4. Version list view (inside Sheet)
- Each row: `border border-outline/10 bg-surface rounded-xl p-4` — identical card style to `ProjectRow` in `project-list.tsx`
- Row columns: [checkbox 44px] [version label + timestamp] [total price] — no "Open" link (we're already in the chat)
- Version label: editable inline on click — identical pattern to `ProjectRow` title editing; `useOptimistic` + `useTransition`
- Timestamp: relative time via `formatRelativeTime` (reuse from `project-list.tsx`)
- Total price: `font-mono font-bold text-sm` right-aligned; `$formatARS(total_price) ARS`
- Auto-label display: when `label` is null, show "Version {N}" in `text-on-surface/50` italic — not editable until user clicks (same click-to-edit UX)
- Checkbox: 20px square, `accent-[#ccff00]` — native checkbox styled via CSS accent-color
- "Compare" CTA: fixed at bottom of sheet; disabled (opacity-40, cursor-not-allowed) unless exactly 2 checkboxes are checked; when active: `bg-primary text-on-primary font-black text-xs uppercase tracking-widest px-6 py-3 rounded-full`
- One-version empty state: when `versions.length === 1`, show a muted notice below the single row: "Run the estimate again to create a second version you can compare." — `text-[11px] text-on-surface/40 text-center mt-4`

### 5. Comparison view (inside Sheet, replaces list)
- Header: back chevron + "Comparing v{A} and v{B}" — 14px bold; back button returns to list view
- Summary row (pinned top of table): Total price delta, price/m2 delta, percentage change — `bg-surface-container-high rounded-lg p-3 mb-4 grid grid-cols-3 gap-2`
- Comparison table: full-width; `border-collapse`; 4 columns: Category | v{A} | v{B} | Delta
- Column widths: Category 40%, v{A} 20%, v{B} 20%, Delta 20%
- Delta cell: signed value with color (red/green/muted) + percentage in parentheses at 11px
- Row striping: odd rows `bg-transparent`, even rows `bg-white/[0.02]` — same as existing category table
- Currency format: `formatARS()` throughout with `$` prefix and `ARS` suffix on header row only

### 6. Loading skeleton
- 3 rows; each `h-14 rounded-xl bg-surface-container animate-pulse` — matches existing animation duration (0.3s)
- No shimmer effect — simple opacity pulse using `animate-pulse` (Tailwind built-in)

## Accessibility Contract

- Sheet must implement focus trap: focus moves into sheet on open; returns to trigger element on close
- Sheet must have `role="dialog"` and `aria-labelledby` pointing to the sheet title
- Checkboxes must have `aria-label="Select version {N} for comparison"`
- "Compare" button when disabled must have `aria-disabled="true"` and `title="Select 2 versions to compare"`
- Inline label edit input: `aria-label="Version label"` + `maxLength={100}`
- Delta values: include visually-hidden sign context — `<span className="sr-only">increase</span>` before positive delta values, `<span className="sr-only">decrease</span>` before negative delta values
- Comparison table: `<caption className="sr-only">Cost comparison between version {A} and version {B}</caption>`
- Sheet backdrop click closes the sheet (keyboard: Escape also closes)
- `prefers-reduced-motion`: slide animation replaced with fade (opacity only) per existing global rule in `globals.css`
