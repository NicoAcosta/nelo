### Project Row Anatomy

Each project row is a horizontal strip:

```
┌────────────────────────────────────────────────────┐
│  [title button — click to edit]    [timestamp]  [Open →] │
│  [edit input — shown when editing]                   │
└────────────────────────────────────────────────────┘
```

- Row height: min 56px (py-4 lg:py-3 — meets 44px touch target requirement)
- Row background: `bg-surface` default, `hover:bg-surface-container` on hover
- Row border: `border border-outline/10 rounded-xl` — matches glass-card language without blur overhead
- Transition: `transition-colors duration-150` on hover

- Title input (edit mode) inherits same size and weight as the display button: `text-sm font-medium`

## Interaction Contract

### State Machine: Project Row

```
DEFAULT
  │
  ├─ hover ──────────────────────► HOVERED (bg-surface-container)
  │                                   │
  │                                   └─ click title button ──► EDITING
  │
  └─ click title button ──────────► EDITING
                                       │
                                       ├─ Escape ──────────────► DEFAULT (value reset)
                                       │
                                       ├─ Enter ───────────────► SAVING
                                       │
                                       └─ blur (onBlur) ────────► SAVING (if value changed)
                                                                      │
                                                                      ├─ success ─────► DEFAULT (optimistic value persisted)
                                                                      │
                                                                      └─ error ───────► DEFAULT (value reverted to original)
```

### Inline Title Edit — Precise Behavior

- **Click target:** `<button>` element with full row width on the left side. NOT a link. Separate "Open" link exists on the right.
- **Edit mode activation:** `setEditing(true)` on click. `autoFocus` on input.
- **Edit mode appearance:** Display button disappears, `<input>` appears in its place. Same font size and weight. `border-b border-primary outline-none` — no border-box, only a bottom underline to signal edit mode. Background transparent (inherits row bg).
- **Edit mode keyboard:** Enter → blur (triggers save). Escape → reset value + exit edit mode without saving.
- **Save trigger:** `onBlur`. If trimmed value equals original, no server call. If different, fire optimistic update + Server Action.
- **Optimistic update:** `useOptimistic` (React 19 stable). Title updates in list immediately. If server returns `{ error }`, revert `setValue` to original title.
- **Max length:** 100 chars. Input has `maxLength={100}` attribute. Server Action validates same limit.
- **Empty value guard:** If input is blank on blur, reset to original. Never save empty title.
- **Click the "Open" button while editing:** Click propagates to blur on input first (onBlur fires, save attempt). Then navigates to `/chat/[id]`. This is acceptable behavior — blur-save before navigation.

### "Open" Button

- Rendered as `<Link href="/chat/{id}">` on the right side of every row.
- Text: "Open" (EN) / "Abrir" (ES)
- Style: `px-4 py-1.5 text-[10px] font-bold uppercase tracking-wider border border-primary/30 rounded-full text-primary hover:bg-primary hover:text-on-primary transition-all active:scale-95`
- Always visible — not hidden on hover reveal. Consistent affordance.

### Sort Order

Projects list sorted by `updated_at DESC` — most recently modified conversation appears first. Renaming a project bumps it (trigger fires on UPDATE). This sort order is enforced in `listProjects()` query, not in the UI.

### Relative Timestamps

- Format: "Today", "Yesterday", "3 days ago", "2 weeks ago", "Mar 15" (date for >30 days)
- Implementation: `Intl.RelativeTimeFormat("en", { numeric: "auto" })` — no library, browser built-in
- Locale-aware: pass locale from `useLocale()` to RelativeTimeFormat constructor
- Rendered in Geist Mono at `text-[11px]` in `text-on-surface/40`

## States

### Loading State

The projects page is a Server Component — data loads on the server before the page renders. No client-side loading spinner needed for initial load.

For the inline rename save in-flight state:
- No spinner. Optimistic update makes it feel instant.
- If there is a visible delay (edge case), the title input briefly shows a slightly faded state via `opacity-70` during `isPending` from `useTransition`.

### Empty State

Shown when `projects.length === 0`. Replaces the project list entirely.

Visual layout:
- Centered vertically in the main content area (not full viewport — just the main section)
- `text-center py-20`
- Icon: `IconEstimates` (existing icon from icons.tsx) at 40x40, `text-on-surface/20`, no background
- Heading: "No projects yet." at `text-sm font-medium text-on-surface/40`
- Body: "Start a conversation to estimate your first project." at `text-sm text-on-surface/30`
- CTA: `<Link href="/chat">` styled as a pill button — `px-5 py-2.5 bg-primary text-on-primary font-bold text-xs uppercase tracking-wider rounded-full hover:brightness-95 transition-all active:scale-95` (matches the "New Estimate" button in header)
- CTA text: "Start your first estimate" (EN) / "Empeza tu primera estimacion" (ES)

Do NOT use an illustration or external image. The icon + text pattern matches the existing chat empty state.

### Error State

If `listProjects()` throws (unexpected — RLS error, network), Next.js error boundary catches it. No custom error UI needed in Phase 11. The App Router default error boundary is sufficient.

If `updateProjectTitle()` Server Action returns `{ error }`:
- Revert the input value to original (`setValue(project.title)`)
- Show inline error message below the input: `text-[11px] text-error` with the error string
- Error clears on next edit attempt (when user clicks title again)
- No toast, no modal — inline is sufficient for a single-field operation

## Animation

Carry forward existing animation primitives from globals.css. No new keyframes for Phase 11.

| Interaction | Animation |
|-------------|-----------|
| Project row mount | `animate-message-in` (0.3s fade + translateY, already defined) — apply to each `<li>` with staggered `animation-delay` (0ms, 50ms, 100ms...) capped at 6 rows for performance |
| Inline edit mode activation | No animation — instant input appearance. Transition only on the underline border: `transition-colors duration-150` |
| "Open" button hover | `transition-all` (existing pattern from header CTA) |
| Row hover | `transition-colors duration-150` |

Stagger cap: apply `animate-message-in` only to the first 6 rows. Rows 7+ render without animation delay to avoid a visually jarring long stagger on users with many projects.

Respect `prefers-reduced-motion`: already handled globally in globals.css with `animation-duration: 0.01ms` override. No Phase 11 work needed.

## Responsive Breakpoints

| Breakpoint | Behavior |
|------------|---------|
| < 768px (mobile) | Sidebar hidden. MobileNav visible at bottom. Page padding px-4. Row padding px-4 py-3. "Open" button text hidden (icon only — use `→` arrow char or just the border pill with no text). Timestamp hidden (show on hover or omit entirely on mobile). |
| >= 768px (md) | Sidebar visible. Header visible. Page padding px-6. Full row layout with timestamp visible. "Open" button text visible. |

Mobile "Open" button collapse: On mobile, replace "Open" text with a right-arrow chevron `→` inside the pill. This preserves the tap target (44px minimum) while reducing horizontal crowding. The chevron is accessible because the `<Link>` has aria-label="Open {project.title}".

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Title button accessible name | `title="Click to rename"` + visible text content |
| Open link accessible name | `aria-label="Open {project.title}"` (not just "Open" — disambiguates for screen readers) |
| Inline input label | No visible label needed (context is clear). Add `aria-label="Project title"` to input. |
| Row list semantics | `<ul>` + `<li>` — do not use `<div>` for the list. Screen readers announce list count. |
| Edit state announcement | When editing starts, input receives focus via `autoFocus`. Screen reader reads label. |
| Timestamp | Wrap in `<time dateTime="{ISO string}">` for machine-readable date. Screen readers read the human label. |
| Error message | `role="alert"` on inline error paragraph — announced immediately by screen readers. |
| Empty state CTA | Regular `<Link>` — no special treatment needed. |
